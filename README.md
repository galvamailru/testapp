# Календарь задач

Приложение для ведения календаря задач с сохранением в PostgreSQL 15.

## Запуск

1. Установите Docker и Docker Compose.
2. Склонируйте репозиторий.
3. Запустите `docker-compose up --build`.
4. Приложение будет доступно по адресу `http://localhost:8085`.

## Структура проекта

- `backend/` - FastAPI приложение
- `frontend/` - HTML, CSS, JS
- `docker-compose.yml` - конфигурация Docker

## API

- `GET /tasks?date=YYYY-MM-DD` - получить задачи на дату
- `POST /tasks` - создать задачу
- `PUT /tasks/{id}` - обновить задачу
- `DELETE /tasks/{id}` - удалить задачу

## Технологии

- **Backend**: Python (FastAPI)
- **Frontend**: HTML, JavaScript (vanilla)
- **Database**: PostgreSQL 15
- **Containerization**: Docker, Docker Compose
- **Port**: 8085

## Функциональные требования

1. Пользователь может просматривать задачи на выбранный день.
2. Пользователь может добавлять новую задачу на определенный день.
3. Пользователь может редактировать существующую задачу.
4. Пользователь может удалять задачу.
5. Задачи отображаются в виде списка по дням календаря.
6. Календарь позволяет переключаться между месяцами.

## Нефункциональные требования

1. Данные сохраняются в PostgreSQL 15.
2. Приложение должно быть контейнеризировано с помощью Docker.
3. Приложение должно быть доступно на порту 8085.
4. Код должен быть модульным и легко расширяемым.
5. Интерфейс должен быть интуитивно понятным.

## Критерии приемки

1. Приложение запускается в Docker и доступно на порту 8085.
2. Пользователь может просматривать задачи на выбранный день.
3. Пользователь может добавлять, редактировать и удалять задачи.
4. Данные сохраняются в PostgreSQL и не теряются после перезапуска контейнеров.
5. Календарь корректно отображает дни и позволяет переключать месяцы.
